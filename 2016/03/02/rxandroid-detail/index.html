响应式编程框架RxAndroid使用简介

RxJava针对Android的定制版本。这个版本中通过增加最少的类使在Android应用中编写响应式组件简单而且无障碍，特别之处在与它还提供了一个Scheduler，可以在主线程或任何给定的Handler上进行调度

github https://github.com/ReactiveX/RxAndroid

总结一下，RxJava让业务逻辑变的更加清晰了，有木有！
到这，作为一个Java程序员可能已经啪啪的在敲键盘了，而Android开发者该疑惑了，说好的AndroidObservable 、AppObservable 、ViewObservable怎么都找不着了，翻看一下源码也没有，why?

这是因为RxAndroid版本升级了，Dan Lew的博客里的介绍对应的是早起的版本，最新版本1.1.0中这些类已经被移除了。RxAndroid项目的核心成员JakeWharton解释说 为了避免RxAndroid被废弃或是成为垃圾场，大多数项目成员开始讨论关于怎样才是这个项目未来最好的方向... （原文）。结果就是RxAndroid功能更加核心化，早期版本里部分已有的功能分离出去成为单独的项目。如RxBinding 、 rx-preferences 、 RxLifecycle。想了解更多可点击这里

二、关于lambda在Android中的使用

很多介绍RxJava的文章会提到lambda.这个RxJava一起使用的确会使整个调用链显的更加简洁，但同时也带了一个问题，对应刚入门的同学来说，代码的可读性将大大降低。Android Studio 中现已对Java代码做类lambda格式化，Android官方目前也没有lambda的支持，所以个人不建议在Android项目中使用lambda,有不同意见欢迎拍砖！
